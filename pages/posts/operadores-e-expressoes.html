Operators and expressions — the math of code

Agora que você já sabe o que são variáveis e tipos de dados, precisa aprender a fazer coisas com eles. É aqui que entram os operadores.

Operadores são símbolos que representam operações: somar, comparar, negar, decidir. Uma expressão é qualquer combinação de variáveis, valores e operadores que produz um resultado.

Operadores aritméticos

Os mesmos da calculadora, com alguns extras:

  • + soma
  • - subtração
  • * multiplicação
  • / divisão
  • % módulo (resto da divisão)

Java:

int a = 10;
int b = 3;

int soma = a + b;          // 13
int resto = a % b;         // 1 (10 / 3 = 3, resta 1)
int divisao = a / b;       // 3 (divisão de inteiros TRUNCA)
double divisaoReal = (double) a / b; // 3.333...

System.out.println(soma);
System.out.println(resto);
System.out.println(divisao);
System.out.println(divisaoReal);

Go:

a := 10
b := 3

soma := a + b             // 13
resto := a % b            // 1
divisao := a / b          // 3 (divisão de inteiros trunca igual)
divisaoReal := float64(a) / float64(b) // 3.333...

fmt.Println(soma)
fmt.Println(resto)
fmt.Println(divisao)
fmt.Println(divisaoReal)

Armadilha: divisão de inteiros

Essa é uma das fontes mais comuns de bug em code review. Se você divide dois inteiros, o resultado é inteiro — a parte decimal é truncada, não arredondada. 10 / 3 dá 3, não 3.333. Se precisa do decimal, converta pelo menos um dos operandos pra double (Java) ou float64 (Go).

Operadores relacionais

Comparam dois valores e devolvem um booleano (true ou false):

  • == igualdade
  • != diferente
  • > maior que
  • < menor que
  • >= maior ou igual
  • <= menor ou igual
int nota = 75;
boolean passou = nota >= 70;  // true
boolean tirouZero = nota == 0; // false

Operadores lógicos

Combinam expressões booleanas. Essenciais para decisões mais complexas:

  • && (AND) — verdadeiro se os dois lados forem verdadeiros
  • || (OR) — verdadeiro se pelo menos um lado for verdadeiro
  • ! (NOT) — inverte o valor booleano

Java:

int idade = 20;
boolean temCarteira = true;

boolean podeDirigir = idade >= 18 && temCarteira;  // true && true = true
boolean eMenor = !(idade >= 18);                    // false
boolean podeOuNao = idade >= 18 || !temCarteira;    // true || false = true

Go:

idade := 20
temCarteira := true

podeDirigir := idade >= 18 && temCarteira  // true
eMenor := !(idade >= 18)                   // false

Short-circuit

Tanto Java quanto Go usam avaliação de curto-circuito. No &&, se a primeira condição é falsa, a segunda nem é avaliada. No ||, se a primeira é verdadeira, a segunda é ignorada. Isso não é só otimização — previne erros: 

// Sem short-circuit, isso daria NullPointerException
String nome = null;
if (nome != null && nome.length() > 0) {
    System.out.println("Válido");
}

Precedência de operadores

Nem toda expressão é lida da esquerda pra direita. Cada operador tem uma prioridade. A boa notícia é que você não precisa decorar a tabela inteira: use parênteses :D 

int resultado = 5 + 3 * 2;        // 11 (multiplicação primeiro)
int comParenteses = (5 + 3) * 2;  // 16 (parênteses primeiro)

boolean confuso = 5 > 3 && 2 < 4 || 1 == 1;
boolean claro = (5 > 3) && ((2 < 4) || (1 == 1));

Qual é mais fácil de entender? Exato. Parênteses não custam nada e evitam suposições.

Exemplo real: validando uma compra

double saldo = 1500.00;
double valorCompra = 1200.00;
boolean clienteVip = true;

boolean podeComprar = (saldo >= valorCompra) && clienteVip;
boolean precisaParcelar = (saldo < valorCompra) && clienteVip;

System.out.println("Pode comprar? " + podeComprar);
System.out.println("Precisa parcelar? " + precisaParcelar);

Resumo

  • Aritméticos: + - * / % — cuidado com divisão de inteiros
  • Relacionais: == != > < >= <= — resultado é booleano
  • Lógicos: && || ! — combinam condições
  • Precedência: parênteses são seus amigos
