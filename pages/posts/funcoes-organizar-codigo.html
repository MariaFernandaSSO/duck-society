Funções — organizando o código para não surtar

Functions — organizing code so you don't lose your mind

Até agora escrevemos tudo dentro do main. Funciona, mas conforme o programa cresce, vira uma sopa de letras. É aí que entram as funções (ou métodos, em Java).

Uma função é um bloco de código nomeado que você pode chamar de vários lugares. Ela recebe valores (parâmetros), faz alguma coisa e pode devolver um resultado (retorno).

Por que usar funções?

  • Reuso: escreve uma vez, usa em vários lugares
  • Organização: cada função faz uma coisa específica
  • Testabilidade: é muito mais fácil testar uma função isolada do que um main gigante
  • Legibilidade: calcularMedia(notas) é mais fácil de entender que 10 linhas de soma e divisão

Sintaxe básica

Java:

// Declaração: [modificador] [tipo retorno] [nome](parâmetros) { corpo }
public static int somar(int a, int b) {
    return a + b;
}

// Chamando
int resultado = somar(5, 3);
System.out.println(resultado); // 8

Go:

// Declaração: func [nome](parâmetros) [tipo retorno] { corpo }
func somar(a int, b int) int {
    return a + b
}

// Chamando
resultado := somar(5, 3)
fmt.Println(resultado) // 8

Parâmetros e retorno

Uma função pode receber zero ou mais parâmetros e retornar zero ou mais valores.

Java:

// Sem parâmetro, sem retorno
public static void dizerOla() {
    System.out.println("Olá!");
}

// Com parâmetro, sem retorno
public static void saudar(String nome) {
    System.out.println("Olá, " + nome + "!");
}

// Com parâmetro, com retorno
public static double calcularMedia(double n1, double n2) {
    return (n1 + n2) / 2;
}

// Sobrecarga: mesmo nome, parâmetros diferentes
public static double calcularMedia(double n1, double n2, double n3) {
    return (n1 + n2 + n3) / 3;
}

Go permite múltiplos retornos (isso é raro em outras linguagens):

// Múltiplos retornos
func dividir(a, b int) (int, int) {
    quociente := a / b
    resto := a % b
    return quociente, resto
}

// Chamando
q, r := dividir(10, 3)
fmt.Println(q, r) // 3 1

Escopo de variáveis

Uma variável declarada dentro de uma função só existe dentro dela. Isso se chama escopo local. Isso é bom: previne que uma função bagunce as variáveis da outra. 

public static void main(String[] args) {
    int x = 10;
    funcao();
    // System.out.println(y); // ERRO: y não existe aqui
}

public static void funcao() {
    int y = 20;
    // System.out.println(x); // ERRO: x não existe aqui
}

O princípio mais importante: faça uma coisa só

A maior lição sobre funções é: uma função deve fazer exatamente uma coisa. Se sua função está "validar e salvar e enviar email", ela está fazendo três coisas. Quebre em três funções.

Um bom sinal é o nome: se você precisa de "e" no nome (validarESalvar), provavelmente são duas funções :D

Exemplo: sistema de notas (sem funções vs com funções)

Sem funções (tudo no main):

public static void main(String[] args) {
    double[] notas = {8.5, 7.0, 9.2, 6.8, 10.0};

    // Calculando média
    double soma = 0;
    for (double nota : notas) {
        soma += nota;
    }
    double media = soma / notas.length;
    System.out.println("Média: " + media);

    // Verificando aprovação
    if (media >= 7.0) {
        System.out.println("Aprovado");
    } else if (media >= 5.0) {
        System.out.println("Recuperação");
    } else {
        System.out.println("Reprovado");
    }

    // Encontrando a maior nota
    double maior = notas[0];
    for (double nota : notas) {
        if (nota > maior) maior = nota;
    }
    System.out.println("Maior nota: " + maior);
}

Com funções:

public static double calcularMedia(double[] notas) {
    double soma = 0;
    for (double nota : notas) {
        soma += nota;
    }
    return soma / notas.length;
}

public static String verificarSituacao(double media) {
    if (media >= 7.0) return "Aprovado";
    if (media >= 5.0) return "Recuperação";
    return "Reprovado";
}

public static double encontrarMaior(double[] notas) {
    double maior = notas[0];
    for (double nota : notas) {
        if (nota > maior) maior = nota;
    }
    return maior;
}

public static void main(String[] args) {
    double[] notas = {8.5, 7.0, 9.2, 6.8, 10.0};

    double media = calcularMedia(notas);
    System.out.println("Média: " + media);
    System.out.println("Situação: " + verificarSituacao(media));
    System.out.println("Maior nota: " + encontrarMaior(notas));
}

Qual dos dois é mais fácil de ler, testar e dar manutenção? O segundo, claramente. Cada função tem um nome que diz exatamente o que ela faz.

Resumo

  • Função = bloco nomeado que pode receber parâmetros e retornar um valor
  • Use funções para organizar, reutilizar e dar nomes significativos ao código
  • Cada função deve fazer uma coisa só
  • Variáveis locais não vazam pra fora da função
  • Java tem sobrecarga (mesmo nome, parâmetros diferentes); Go tem múltiplos retornos
