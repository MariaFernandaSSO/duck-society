Loops: for, while and do-while — the art of repeating without going crazy

O computador não reclama de repetir a mesma tarefa 1.000 vezes. Você, como ser humano, provavelmente vai reclamar na 5ª. É por isso que existem loops (também chamados de estruturas de repetição).

A ideia é simples: em vez de escrever a mesma instrução 20 vezes, você escreve uma vez e diz quantas vezes repetir :D

for — quando você sabe quantas vezes repetir

O for é o loop mais comum. Você define uma variável de controle, uma condição de parada e como a variável muda a cada volta.

Java:

// Imprime de 0 a 4
for (int i = 0; i < 5; i++) {
    System.out.println("Volta " + i);
}

// Percorrendo um array
String[] nomes = {"Mari", "João", "Ana"};
for (int i = 0; i < nomes.length; i++) {
    System.out.println(nomes[i]);
}

// for-each (quando não precisa do índice)
for (String nome : nomes) {
    System.out.println(nome);
}

Go:

// Imprime de 0 a 4 (Go só tem for, nada de while)
for i := 0; i < 5; i++ {
    fmt.Println("Volta", i)
}

// Percorrendo um slice
nomes := []string{"Mari", "João", "Ana"}
for i := 0; i < len(nomes); i++ {
    fmt.Println(nomes[i])
}

// range (equivalente ao for-each)
for i, nome := range nomes {
    fmt.Println(i, nome)
}

Repare uma diferença: em Go, i e nome são declarados dentro do for. Em Java, o int i também fica dentro do escopo do loop. As variáveis declaradas ali não existem fora do loop.

while — quando você NÃO sabe quantas vezes repetir

O while repete enquanto uma condição for verdadeira. Se a condição já começar falsa, o bloco nunca executa.

Java:

// Lendo caracteres até o usuário digitar 'sair'
Scanner scanner = new Scanner(System.in);
String entrada = "";

while (!entrada.equals("sair")) {
    System.out.print("Digite algo (ou 'sair' pra terminar): ");
    entrada = scanner.nextLine();
    System.out.println("Você digitou: " + entrada);
}

Go:

// Em Go, while é escrito com for mesmo
reader := bufio.NewReader(os.Stdin)
entrada := ""

for entrada != "sair" {
    fmt.Print("Digite algo (ou 'sair' pra terminar): ")
    entrada, _ = reader.ReadString('\n')
    entrada = strings.TrimSpace(entrada)
    fmt.Println("Você digitou:", entrada)
}

do-while — executa pelo menos uma vez

O do-while é igual ao while, mas a condição é verificada depois de executar o bloco. Garante que o bloco rode pelo menos uma vez.

Java:

Scanner scanner = new Scanner(System.in);
int senha;

do {
    System.out.print("Digite a senha: ");
    senha = scanner.nextInt();
} while (senha != 1234);

System.out.println("Acesso liberado!");

Go não tem do-while. Você simula com um for infinito e um break: 

senha := 0

for {
    fmt.Print("Digite a senha: ")
    fmt.Scan(&senha)
    if senha == 1234 {
        break
    }
}

fmt.Println("Acesso liberado!")

Exercício mental: qual usar?

  • for — quando você tem um número definido de repetições (percorrer array, contar até 10)
  • while — quando a repetição depende de uma condição que pode não acontecer (esperar entrada do usuário, processar fila)
  • do-while — quando precisa garantir pelo menos uma execução (mostrar menu, pedir confirmação)

Como evitar loop infinito

Loop infinito é quando a condição de parada nunca se torna falsa. Seu programa trava, o navegador congela, você força o reboot. Causas comuns:

  • Esquecer de incrementar a variável de controle: for (int i = 0; i < 10; )
  • Condição que nunca muda: while (true) sem break
  • Comparação com = em vez de == (mais comum em C, em Java isso dá erro de compilação)

Exemplo real: calculando média da turma

double[] notas = {8.5, 7.0, 9.2, 6.8, 10.0};
double soma = 0;

for (int i = 0; i < notas.length; i++) {
    soma += notas[i];
}

double media = soma / notas.length;
System.out.println("Média da turma: " + media);

// Com for-each fica mais limpo
soma = 0;
for (double nota : notas) {
    soma += nota;
}
media = soma / notas.length;
System.out.println("Média (com for-each): " + media);

Resumo

  • for: quando você sabe o número de repetições (ou tem um índice)
  • while: quando repete enquanto uma condição for verdadeira
  • do-while: igual ao while, mas roda pelo menos uma vez (não existe em Go)
  • Sempre garanta que a condição de parada vai ser alcançada
  • Prefira for-each (Java) ou range (Go) para percorrer coleções
