Algoritmos

O que é um algoritmo (e por que você já usa sem saber)

What is an algorithm (and why you already use one without knowing)

Se você nunca escreveu uma linha de código na vida e ouviu a palavra "algoritmo", provavelmente pensou em coisa de gênio da computação, fórmulas complexas ou algo que só pessoas com óculos fundo de garrafa entendem. A verdade é bem mais simples: você segue algoritmos o tempo todo.

O que é um algoritmo, afinal?

Um algoritmo é uma sequência finita de passos que resolve um problema. Só isso. Não precisa de computador, não precisa de código, não precisa de nada além de uma cabeça funcionando.

Vou te mostrar três exemplos que você vive sem perceber:

1. Receita de bolo

Pegar ingredientes → misturar → colocar no forno → esperar → servir. Se você seguir os passos na ordem certa, o bolo sai. Se pular etapas (colocar no forno antes de misturar?), o bolo não sai.

2. GPS traçando rota

Ponto A → ponto B. O GPS testa várias rotas, calcula o tempo de cada uma, escolhe a mais rápida. Isso é um algoritmo de busca executado milhares de vezes por segundo.

3. Fila do banco

Chegar → pegar senha → esperar → ser chamado → ser atendido → sair. É uma fila (estrutura que vamos estudar mais pra frente) com regras claras de quem vem depois de quem.

O algoritmo mais famoso do mundo

Você já fez "para cada item da lista, compare com o que estou procurando". Parabéns :D você usou busca linear. Se já abriu o meio de um livro e decidiu pra que lado ir baseado na página, usou busca binária.

A diferença entre um algoritmo e "só fazer as coisas" é que um algoritmo é exato, finito e determinístico. Dada a mesma entrada, ele sempre produz a mesma saída.

Como representamos algoritmos?

Tem duas formas principais antes de chegar no código:

  • Fluxograma: desenhar quadrados e setas. Ótimo pra visualizar, péssimo pra compartilhar.
  • Pseudocódigo: uma linguagem intermediária, parece código mas não é. É o que mais usamos pra planejar.

Exemplo de pseudocódigo pra somar dois números: 

INICIO
  LER numero1
  LER numero2
  resultado = numero1 + numero2
  MOSTRAR resultado
FIM

Mesmo algoritmo, duas linguagens

Uma das coisas mais legais de algoritmo é que ele independe da linguagem. A lógica é a mesma; muda só a sintaxe. Veja como implementar um algoritmo que verifica se um número é par em Java e em Go:

Java:

public class ParOuImpar {
    public static void main(String[] args) {
        int numero = 7;

        if (numero % 2 == 0) {
            System.out.println("Par");
        } else {
            System.out.println("Ímpar");
        }
    }
}

Go:

package main

import "fmt"

func main() {
    numero := 7

    if numero % 2 == 0 {
        fmt.Println("Par")
    } else {
        fmt.Println("Ímpar")
    }
}

Percebeu? A lógica é idêntica: pegar um número, testar se o resto da divisão por 2 é zero, decidir o que mostrar. A única coisa que muda é o jeito de escrever.

Por que isso importa?

Porque entender que algoritmo é sobre lógica, não sobre linguagem muda completamente como você aprende programação. Você não precisa aprender 15 linguagens; você precisa aprender a pensar em algoritmos. Depois, é só traduzir pra sintaxe que o cliente pediu :D
